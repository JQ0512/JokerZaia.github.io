  • C风格的编程模式
  • 面向对象的OOP模式

  1. C++的发展历史
    首先，C++由C语言发展而来。兼具低级语言的效率和高级语言的通用性可移植性。
    为了应对大型程序编写时C编程遇到的困难，设计了面向对象编程原理（object oriented programming)，将数据和方法以类的形式定义，有利于创建可以重用的代码。
    另外，C++还提供了另一种风格去编写程序，那就是泛型编程。它与OOP目标相同，就是使得抽象的数据更加简单。不过泛型编程更加侧重于特定的数据类型，提供常见任务（如排序）等的特殊工具。

  3. 程序设计的技巧
  • 编写程序
  • 编译源代码

第二章.开始学习C++

  1. C++的编程格式
      C语言使用scanf-printf等一些函数来实现输入输出。而在C++中，我们通常使用cin,cout实现输出。当然，C语言里面的printf,scanf同样是可以用的（只要用了对应的库）。一个一般的C++程序里应该包括如下内容：
  • 注释，用//
  • 预处理编译指令#include
  • 函数头（返回类型，名称，参数列表）
  • 编译指令using namespace
  • 函数主体内容
  • 返回，return 语句

C++代码中有且仅有一个main函数，而且不能被其他函数调用。
  iostream和头文件同样很重要。与C语言类似，头文件里面包括一些基本的函数供我们调用，而iostream（输入输出流）控制的是输入输出等最基本最核心的语句，所以不可没有。当然，C++也是可以用C头文件的。不过不像C的stdio.h，C++头文件很多不以后缀.h命名。有些是直接用C文件，有些自己弄，还有些就干脆把.h换成c***来使用，比如就把math.h变成cmath（这个头文件里面放的全是数学函数）。另外，编译指令using在C++中必不可少，有助于组织程序（还可以把多个厂商现有的代码组合起来虽然我不会）。按照这种方式，类，函数，变量是C++编译器的标准组件，它们都被放在了名称空间std中
2. cout和cin的用法

cin>>******;
cout<<******<<endl;
  1. 声明变量和赋值，cout的拼接操作
  2. cout竟然是一个类的对象
      cout是类ostream中的一个对象。istream和ostream输入输出类都来自iostream中定义的类库。我们并未编写它，但是我们可以用它。
  3. 函数的编写与调用
      调用函数的基本格式像这个样子：
x=sqrt(6.25);

x是赋值对象，存放函数的返回值。sqrt是函数名称，6.25是函数的参数，是个浮点数。最后每一条语句都别忘记了加上分号。哦对了，可能有些函数没有返回值，这也很正常。那么，就请不要赋值了，直接用函数体里面执行的过程就可以咯。
  光靠头文件里面提供的一些函数绝对是不够用的，实际开发中我们会经常性的编写自己的函数（毕竟头文件里面的函数也是人编出来的嘛）。当我们自定义函数的时候，我们是先声明再编写。也就是说如果不急着在程序开始写函数内容，我们通常会先放一个函数的signature上面，到了后面我们再进行编写。一个函数的signature包括返回值，名称与参数列表。编写函数就像这个样子：
